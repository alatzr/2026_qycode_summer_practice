## 2. 栈：为什么括号题离不开它

### 2.1 栈的直觉

栈是“后进先出”。

括号嵌套也是后开的括号先关闭：

```text
([{}])
```

匹配顺序：

```text
{ } 最先匹配
[ ] 第二匹配
( ) 最后匹配
```

这正好对应：

```text
左括号入栈
右括号匹配栈顶
```

### 2.2 C++ 栈操作

```cpp
stack<char> st;
st.push('(');   // 入栈
st.top();       // 查看栈顶
st.pop();       // 弹出栈顶
st.empty();     // 判断是否为空
```

教师必须强调：

```cpp
if(!st.empty() && st.top() == '('){
    // 先判断非空，再访问 top
}
```

不能写成：

```cpp
if(st.top() == '(' && !st.empty()){
    // 错：空栈时 top() 已经越界
}
```

### 2.3 一种括号可以不用栈

如果只有 `(` 和 `)`，可以用 `bal`：

```text
遇到 '('，bal++
遇到 ')'，bal--
过程中 bal 不能小于 0
最后 bal 必须等于 0
```

代码：

```cpp
bool check(string s){
    int bal = 0;
    for(char c: s){
        if(c == '(') bal++;
        else bal--;

        if(bal < 0) return false; // 右括号太多
    }
    return bal == 0;              // 左右数量也要相等
}
```

但只要有多种括号，计数就不够。

反例：

```text
([)]
```

`()` 和 `[]` 数量都对，但顺序错了，必须用栈。

## 3. 612C：多种括号替换计数

### 3.1 题目结构

给一个只含括号字符的字符串，括号类型包括：

```text
() [] {} <>
```

如果一个右括号类型不对，可以替换成正确类型。问最少替换次数。如果无法变成合法括号序列，输出 `Impossible`。

这题的重点：

```text
类型错可以替换，结构错不能替换。
```

### 3.2 什么叫结构错

结构错的两种情况：

1. 右括号出现时，栈为空。
2. 扫描结束后，栈不为空。

例 1：

```text
)] 
```

第一个字符就是右括号，没有左括号可匹配，无法修。

例 2：

```text
([]
```

最后还有一个 `(` 没有被匹配，无法修。

### 3.3 什么叫类型错

例：

```text
([)]
```

手动模拟：

| 字符 | 栈 | 操作 | 替换次数 |
|---|---|---|---:|
| `(` | `(` | 左括号入栈 | 0 |
| `[` | `([` | 左括号入栈 | 0 |
| `)` | `(` | 栈顶是 `[`，应该配 `]`，类型错 | 1 |
| `]` | 空 | 栈顶是 `(`，应该配 `)`，类型错 | 2 |

可以把第 3 个右括号 `)` 换成 `]`，第 4 个右括号 `]` 换成 `)`，所以答案是 2。

### 3.4 匹配函数设计

先写两个小函数：

```cpp
bool isleft(char c){
    return c == '(' || c == '[' || c == '{' || c == '<';
}

char need(char c){
    if(c == '(') return ')';
    if(c == '[') return ']';
    if(c == '{') return '}';
    return '>';
}
```

讲解：

- `isleft` 判断是不是左括号。
- `need` 返回某个左括号需要的右括号。

### 3.5 612C 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

bool isleft(char c){
    return c == '(' || c == '[' || c == '{' || c == '<';
}

char need(char c){
    if(c == '(') return ')';
    if(c == '[') return ']';
    if(c == '{') return '}';
    return '>';
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;

    stack<char> st; // 存还没有匹配的左括号
    int ans = 0;    // 需要替换的右括号数量
    bool ok = true;

    for(char c: s){
        if(isleft(c)){
            st.push(c); // 左括号等待后面的右括号匹配
        }else{
            if(st.empty()){
                ok = false; // 没有左括号能和它匹配
                break;
            }

            char l = st.top();
            st.pop();

            if(need(l) != c){
                ans++; // 结构能匹配，但右括号类型不对，需要替换
            }
        }
    }

    if(!st.empty()) ok = false; // 还有左括号没闭合

    if(!ok) cout << "Impossible\n";
    else cout << ans << '\n';

    return 0;
}
```

复杂度：`O(n)`。

### 3.6 教师提问

问：

```text
输入：([)]
输出是多少？
```

答：2。

问：

```text
输入：[(])
输出是多少？
```

让学生自己模拟：左括号入栈，右括号匹配栈顶，只要栈不空，类型错就加 1。

问：

```text
输入：())(
为什么不是改两个括号就行？
```

答：本题只能替换右括号类型，不能把右括号变左括号，也不能调整顺序。结构已经错了。

## 4. 223A：合法括号子串与区间贡献

### 4.1 题目结构

给一个只含 `(`、`)`、`[`、`]` 的字符串，找一个合法括号子串，使其中 `[` 数量最多，并输出：

1. 最大 `[` 数量；
2. 一个对应的子串。

这题比普通括号匹配多一步：

```text
不仅要知道哪里合法，还要统计合法区间中有多少个 '['。
```

### 4.2 栈里为什么存位置

612C 只需要判断类型，所以栈里存字符。

223A 需要输出子串范围，所以栈里要存下标。

思路：

- 左括号位置入栈；
- 右括号能匹配栈顶左括号时，弹出；
- 弹出后，当前合法后缀的左边界是 `st.top()+1`；
- 用前缀和快速统计这个区间中 `[` 的数量。

### 4.3 前缀数组设计

定义：

```text
pre[i] = 字符串前 i 个字符中 '[' 的数量
```

字符串下标 0 开始，前缀数组下标 1 开始。

区间 `[l,r]` 中 `[` 数量：

```cpp
pre[r + 1] - pre[l]
```

### 4.4 手动模拟

字符串：

```text
([])[]
```

| i | 字符 | 栈变化 | 当前合法区间 | `[` 数量 |
|---:|---|---|---|---:|
| 初始 |  | `[-1]` | 无 | 0 |
| 0 | `(` | `[-1,0]` | 无 | 0 |
| 1 | `[` | `[-1,0,1]` | 无 | 0 |
| 2 | `]` | `[-1,0]` | `[1,2]` 即 `[]` | 1 |
| 3 | `)` | `[-1]` | `[0,3]` 即 `([])` | 1 |
| 4 | `[` | `[-1,4]` | 无 | 1 |
| 5 | `]` | `[-1]` | `[0,5]` 即 `([])[]` | 2 |

答案是：

```text
2
([])[]
```

### 4.5 223A 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

bool isleft(char c){
    return c == '(' || c == '[';
}

bool match(char l, char r){
    return (l == '(' && r == ')') || (l == '[' && r == ']');
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;
    int n = (int)s.size();

    vector<int> pre(n + 1, 0);
    for(int i = 0; i < n; i++){
        pre[i + 1] = pre[i] + (s[i] == '['); // 统计前 i+1 个字符中的 [
    }

    stack<int> st;
    st.push(-1); // 哨兵，表示当前合法段左边界前一位

    int best = 0; // 最多的 [ 数量
    int L = 0, R = -1; // 最优子串范围；R=-1 表示当前没有答案

    for(int i = 0; i < n; i++){
        if(isleft(s[i])){
            st.push(i); // 左括号下标入栈
        }else{
            if(st.top() != -1 && match(s[st.top()], s[i])){
                // 当前右括号能和栈顶左括号匹配
                st.pop();

                int l = st.top() + 1;             // 当前合法后缀左端点
                int cnt = pre[i + 1] - pre[l];    // 合法区间内 [ 的数量

                if(cnt > best){
                    best = cnt;
                    L = l;
                    R = i;
                }
            }else{
                // 无法匹配，这个位置成为新的非法边界
                st.push(i);
            }
        }
    }

    cout << best << '\n';
    if(best > 0){
        for(int i = L; i <= R; i++) cout << s[i];
    }
    cout << '\n';

    return 0;
}
```

### 4.6 易错点

- 栈里存的是位置，不是字符。
- `-1` 是哨兵，不能拿它访问 `s[-1]`。
- 比较的是 `[` 数量，不是子串长度。
- 区间统计公式是 `pre[r+1]-pre[l]`。

## 5. 1175B：程序模拟与溢出保护

### 5.1 题目结构

题目给一段小程序，指令有三类：

```text
add
for x
end
```

含义：

- `add`：答案加上当前循环倍数；
- `for x`：进入一层循环，后面的指令要执行 `x` 倍；
- `end`：退出当前循环层。

需要判断答案是否超过 `2^32-1`。一旦超过，输出：

```text
OVERFLOW!!!
```

否则输出最终答案。

### 5.2 当前倍数的含义

例：

```text
for 2
  add
  for 3
    add
  end
end
```

模拟：

| 指令 | 当前倍数 | 答案 |
|---|---:|---:|
| 初始 | 1 | 0 |
| `for 2` | 2 | 0 |
| `add` | 2 | 2 |
| `for 3` | 6 | 2 |
| `add` | 6 | 8 |
| `end` | 2 | 8 |
| `end` | 1 | 8 |

当前倍数等于所有未结束循环次数的乘积。

### 5.3 为什么用栈

进入一层循环时，当前倍数变大。

退出时，要恢复上一层倍数。

这正好是栈：

```text
for x -> push(上一层倍数 * x)
end   -> pop()
add   -> ans += top()
```

### 5.4 溢出不能事后判断

错误写法：

```cpp
cur = cur * x;
if(cur > LIM) cur = LIM;
```

如果 `cur*x` 已经超过 `long long`，结果可能已经错了。

稳妥写法：

```cpp
if(cur > LIM / x) nxt = LIM;
else nxt = cur * x;
```

这里 `LIM = 2^32`。只要答案达到 `LIM`，就说明超过了 `2^32-1`。

### 5.5 1175B 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

const long long LIM = (1LL << 32); // 超过 2^32-1 等价于达到 2^32

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    stack<long long> st;
    st.push(1); // 初始没有循环，当前倍数是 1

    long long ans = 0;

    for(int i = 1; i <= n; i++){
        string op;
        cin >> op;

        if(op == "add"){
            ans += st.top(); // 当前 add 会执行 st.top() 次

            if(ans >= LIM){
                cout << "OVERFLOW!!!\n";
                return 0;
            }
        }else if(op == "for"){
            long long x;
            cin >> x;

            long long cur = st.top();
            long long nxt;

            // 先判断再乘，防止乘法本身溢出
            if(cur > LIM / x) nxt = LIM;
            else nxt = cur * x;

            // 超过上限后截断到 LIM 即可，不必保存真实大数
            if(nxt > LIM) nxt = LIM;
            st.push(nxt);
        }else{
            st.pop(); // end：退出当前循环层
        }
    }

    cout << ans << '\n';
    return 0;
}
```

复杂度：`O(n)`。

### 5.6 教师提问

问：

```text
为什么上限用 2^32，而不是 2^32-1？
```

答：合法最大值是 `2^32-1`，所以一旦 `ans >= 2^32` 就溢出。

问：

```text
倍数已经超过上限后，为什么可以只保存 LIM？
```

答：后面只关心是否会导致答案溢出，不需要真实倍数。保存 `LIM` 能防止继续乘爆。

## 6. 34C：页码解析与连续段压缩

### 6.1 题目结构

输入是一行逗号分隔的页码，例如：

```text
1,2,3,1,1,2,6,6,2
```

需要去掉重复页码，并把连续页码压缩成区间：

```text
1-3,6
```

### 6.2 解题步骤

1. 扫描字符串，把逗号分隔的数字解析出来。
2. 排序。
3. 去重。
4. 从左到右合并连续段。
5. 按格式输出。

### 6.3 手动模拟

原始页码：

```text
1,2,3,1,1,2,6,6,2
```

排序去重后：

```text
1 2 3 6
```

连续段：

| 起点 | 终点 | 输出 |
|---:|---:|---|
| 1 | 3 | `1-3` |
| 6 | 6 | `6` |

最终：

```text
1-3,6
```

### 6.4 34C 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;

    vector<int> a;
    int x = 0; // 当前正在解析的数字

    for(int i = 0; i <= (int)s.size(); i++){
        if(i == (int)s.size() || s[i] == ','){
            a.push_back(x); // 一个数字解析完成
            x = 0;
        }else{
            x = x * 10 + (s[i] - '0');
        }
    }

    sort(a.begin(), a.end());
    a.erase(unique(a.begin(), a.end()), a.end()); // 去重

    vector<pair<int,int>> seg;

    for(int i = 0; i < (int)a.size(); ){
        int l = a[i], r = a[i];
        int j = i + 1;

        while(j < (int)a.size() && a[j] == r + 1){
            r = a[j];
            j++;
        }

        seg.push_back({l, r});
        i = j;
    }

    for(int i = 0; i < (int)seg.size(); i++){
        if(i) cout << ',';

        int l = seg[i].first;
        int r = seg[i].second;

        if(l == r) cout << l;
        else cout << l << '-' << r;
    }
    cout << '\n';

    return 0;
}
```

### 6.5 易错点

- 扫描到字符串末尾也要结算最后一个数字。
- 必须去重，否则 `1,1,2` 会影响连续段判断。
- 单点区间输出 `x`，不是 `x-x`。
- 逗号只能在段与段之间输出，不能多输出最后一个逗号。

## 7. 727B：金额字段解析

### 7.1 本题训练重点

这类题的核心不是金额本身，而是：

```text
从字母、数字、符号混合的字符串中，提取出数值字段，并按题面规则解释。
```

课堂上要训练学生先读清楚题面的格式规则：

- 哪些字符表示商品名？
- 哪些字符属于价格？
- `.` 是小数点还是分组符？
- 输出是否需要补零？
- 总和是否可能超过 `int`？

### 7.2 通用扫描框架

```cpp
for(int i = 0; i < (int)s.size(); ){
    // 先跳过商品名或普通字母段
    while(i < (int)s.size() && islower(s[i])) i++;

    // 再收集价格字段
    string price;
    while(i < (int)s.size() && (isdigit(s[i]) || s[i] == '.')){
        price += s[i];
        i++;
    }

    // 对 price 按题面规则解析
}
```

教师强调：

不要一边扫一边乱加。先把字段边界找出来，再解析字段，会更稳。

### 7.3 金额转最小单位

很多金额题适合把金额统一转成“分”或最小货币单位。

例：

```text
12.30 -> 1230
5     -> 500
0.07  -> 7
```

这样可以避免浮点数误差。

不要写：

```cpp
double x;
```

金额题尽量用整数。

### 7.4 解析函数示意

下面代码展示“按最后一个点判断是否有两位小数”的思路。具体比赛时要以题面规则为准。

```cpp
long long parseMoney(string s){
    int n = (int)s.size();
    int last = -1;

    for(int i = 0; i < n; i++){
        if(s[i] == '.') last = i;
    }

    long long dollar = 0;
    long long cent = 0;

    int endDollar = n;
    if(last != -1 && n - last - 1 == 2){
        // 最后一个点后面正好两位，按小数处理
        endDollar = last;
        cent = (s[last + 1] - '0') * 10 + (s[last + 2] - '0');
    }

    for(int i = 0; i < endDollar; i++){
        if(s[i] != '.'){
            dollar = dollar * 10 + (s[i] - '0'); // 忽略分组点
        }
    }

    return dollar * 100 + cent;
}
```

### 7.5 讲题注意

这道题要让学生形成习惯：

```text
格式题先写解析器，再写业务逻辑。
```

如果题面规定和上面的示意不同，就只替换 `parseMoney`，主扫描框架不变。

## 8. 41C：邮箱描述还原

### 8.1 本题训练重点

这类题不是简单判断邮箱是否合法，而是从一种“被读出来的字符串”还原真正邮箱。

常见规则是：

```text
at  -> @
dot -> .
```

但不是所有 `at`、`dot` 都能替换，因为替换后必须满足邮箱格式。

### 8.2 课堂模型

做这类构造题时，不要贪心乱替换，先分清：

1. 哪一个 `at` 应该变成唯一的 `@`；
2. 哪些 `dot` 可以变成 `.`；
3. 替换后开头、结尾、`@` 左右是否为空；
4. 如果有多个候选，题面要求如何选择。

### 8.3 枚举 `at` 的位置

因为邮箱通常只能有一个 `@`，所以可以枚举每个 `"at"`：

```cpp
vector<int> pos;
for(int i = 0; i + 1 < (int)s.size(); i++){
    if(s.substr(i, 2) == "at"){
        pos.push_back(i);
    }
}
```

对每个候选位置构造一个答案，再检查是否合法。

### 8.4 合法性检查函数

```cpp
bool good(string t){
    if(t.empty()) return false;
    if(t[0] == '.' || t[0] == '@') return false;
    if(t.back() == '.' || t.back() == '@') return false;

    int cnt = 0;
    for(char c: t){
        if(c == '@') cnt++;
    }

    return cnt == 1; // 恰好一个 @
}
```

教师提醒：

完整合法性要按题面补充。例如有的题还要求 `@` 两边不能有连续点，或者域名段不能为空。

### 8.5 易错点

- 不是所有 `at` 都能替换。
- `dot` 在开头或结尾通常不能替换成 `.`。
- 构造题要先保证合法，再考虑最优。
- 如果题面要求“最短”或“字典序”，比较函数要单独写清楚。

## 11. 本课总易错点清单

### 11.1 栈

- 空栈时不能访问 `top()`。
- 左括号入栈，右括号才匹配。
- 多种括号不能只用计数。

### 11.2 括号子串

- 需要输出区间时，栈里存位置。
- 哨兵 `-1` 只表示边界，不能访问 `s[-1]`。
- 223A 统计的是 `[` 数量，不是长度。

### 11.3 程序模拟

- `for` 入栈，`end` 出栈。
- `add` 加的是当前倍数。
- 达到 `2^32` 就溢出。
- 乘法要先判断再乘。

### 11.4 字符串解析

- 扫描到结尾也要结算最后一段。
- 输出分隔符不能多也不能少。
- 格式题要按题面写解析器，不要凭直觉。

## 14. 本节总结

- 括号题的第一反应是栈。
- 如果只判断合法性，栈里可以存字符；如果要输出区间，栈里要存位置。
- 表达式和格式题要先拆字段，再处理字段。
- 程序模拟题要保存层级状态。
- 溢出题要提前截断，不要等溢出后再判断。
