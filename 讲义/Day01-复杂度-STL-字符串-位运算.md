## 2. 复杂度：先判断能不能跑

### 2.1 为什么复杂度是第一步

教师强调：

同样是“枚举”，`n <= 20` 和 `n <= 2e5` 完全不是一类题。竞赛里大量错误不是因为不会写，而是因为一开始选了一个必超时的方向。

粗略估算：

| 数据范围 | 可接受复杂度 | 典型做法 | 课堂解释 |
|---:|---|---|---|
| `n <= 20` | `O(2^n)` | 子集枚举、状态压缩 | 20 位以内二进制枚举还可做 |
| `n <= 100` | `O(n^3)` | Floyd、三重循环 DP | `1e6` 级别，通常安全 |
| `n <= 2000` | `O(n^2)` | 双重循环、二维 DP | `4e6` 级别，通常安全 |
| `n <= 5000` | 勉强 `O(n^2)` | 简单双循环 | `2.5e7`，看常数 |
| `n <= 2e5` | `O(n log n)` 或 `O(n)` | 排序、二分、前缀和、扫描 | 普及/提高常见范围 |
| `n <= 1e6` | `O(n)` | 线性扫描、计数 | 尽量不做复杂容器嵌套 |

注意：这里是估算，不是死规则。多组数据时要看总量。

### 2.2 课堂提问

给学生 30 秒判断：

```text
n <= 2e5，q <= 2e5，每次询问都扫一遍区间，可以吗？
```

引导答案：

一次最坏 `O(n)`，总共 `O(nq)=4e10`，不可以。要预处理。

再问：

```text
如果预处理 O(n)，每次询问 O(1)，总复杂度是多少？
```

答案：`O(n+q)`。

这就是 313B 的核心。

### 2.3 两层循环不一定都是 `O(n^2)`

板书代码：

```cpp
for(int i = 1; i <= n; i++){
    for(int j = i; j <= n; j += i){
        cnt++;
    }
}
```

讲解：

内层不是每次跑 `n`，而是：

```text
i=1 跑 n 次
i=2 跑 n/2 次
i=3 跑 n/3 次
...
i=n 跑 1 次
```

总次数约为：

```text
n/1 + n/2 + n/3 + ... + n/n = n(1 + 1/2 + 1/3 + ... + 1/n)
```

约为 `O(n log n)`。

教师提醒：复杂度分析要看循环变量怎么变，不要只数有几层循环。

### 2.4 本节题目复杂度预期

| 题目 | 目标复杂度 | 不能接受的写法 |
|---|---:|---|
| 1023A | `O(n+m)` | 枚举 `*` 匹配长度再反复构串 |
| 723B | `O(n)` | 先拆大量子串再逐个处理 |
| 1042B | `O(n+8^3)` | 枚举所有购买子集 `O(2^n)` |
| 313B | `O(n+q)` | 每个询问扫 `[l,r]` |
| 90B | `O(nm)` | 每个格子再扫整行整列 |
| 166A | `O(n log n)` | 手写复杂排名模拟 |

## 4. STL：用库，但要知道复杂度

### 4.1 `vector`：动态数组

用途：存一批数量不固定的数据。

常用操作：

| 操作 | 含义 | 复杂度 |
|---|---|---:|
| `v.push_back(x)` | 末尾加入元素 | 均摊 `O(1)` |
| `v.pop_back()` | 删除末尾元素 | `O(1)` |
| `v[i]` | 访问第 `i` 个元素 | `O(1)` |
| `v.size()` | 元素个数 | `O(1)` |
| `sort(v.begin(), v.end())` | 排序 | `O(n log n)` |

示例代码：

```cpp
vector<int> v;
for(int i = 1; i <= n; i++){
    int x;
    cin >> x;
    v.push_back(x); // 不知道最终有多少元素时，直接追加
}

sort(v.begin(), v.end()); // 从小到大排序
```

注意：

```cpp
int len = (int)v.size();
```

`v.size()` 不是 `int`，和 `int` 混用时最好强转，避免比较时出现奇怪警告或边界问题。

### 4.2 `string`：字符串也是数组

常用操作：

| 操作 | 含义 |
|---|---|
| `s.size()` | 长度 |
| `s[i]` | 第 `i` 个字符，0 下标 |
| `s.substr(pos, len)` | 从 `pos` 开始取长度为 `len` 的子串 |
| `s += c` | 末尾追加字符 |
| `s == t` | 判断两个字符串是否相同 |

课堂重点：`substr(pos, len)` 的第二个参数是长度，不是右端点。

示例：

```cpp
string s = "abcdef";
cout << s.substr(2, 3) << '\n'; // 输出 cde
```

### 4.3 `pair` 与结构体排序

`pair<int,int>` 默认先比较 `first`，再比较 `second`。

但竞赛排名往往不是自然顺序，例如：

- 解题数高的在前；
- 解题数相同，罚时低的在前。

这时建议写结构体和比较函数。

```cpp
struct Node{
    int p, t; // p: 解题数，t: 罚时
};

bool cmp(Node a, Node b){
    if(a.p != b.p) return a.p > b.p; // 解题数高在前
    return a.t < b.t;                // 解题数相同，罚时低在前
}
```

教师强调：

比较函数必须回答“a 是否应该排在 b 前面”。不要写 `>=` 或 `<=`，否则可能破坏排序规则。

### 4.4 `map`、`set` 与数组计数

`map`：键到值的映射。

```cpp
map<string, int> mp;
mp["abc"]++; // 字符串 abc 出现次数加 1
```

`set`：自动去重，并保持有序。

```cpp
set<int> st;
st.insert(5);
st.insert(2);
st.insert(5); // 第二个 5 不会重复加入
```

复杂度一般是 `O(log n)`。

如果字符只在 `'a'` 到 `'z'`，用数组更快更清楚：

```cpp
int cnt[26] = {};
for(char c: s){
    cnt[c - 'a']++; // 字符映射到 0..25
}
```

教师提醒：

STL 是工具，不是万能药。能用数组计数时，不要强行 `map`。

## 5. 模型一：字符串扫描

### 5.1 一句话核心

字符串扫描题的核心是：从左到右看字符，用少量变量维护“当前状态”，遇到分隔符就结算。

板书：

```text
扫描字符 -> 更新状态 -> 遇分隔符 -> 结算当前段 -> 清空临时变量
```

### 5.2 什么叫状态

以 723B 为例：

题目给一个字符串，包含：

- 字母；
- 下划线 `_`；
- 左括号 `(`；
- 右括号 `)`。

需要求：

1. 括号外最长单词长度；
2. 括号内单词数量。

关键状态：

| 变量 | 含义 |
|---|---|
| `in` | 当前是否在括号内 |
| `len` | 当前连续字母段长度 |
| `mx` | 括号外最长单词长度 |
| `cnt` | 括号内单词数量 |

### 5.3 手动模拟

例子：

```text
ab_cd(ef_g)h
```

扫描过程：

| 位置 | 字符 | 当前状态 | 操作 |
|---:|---|---|---|
| 1 | `a` | 括号外 | `len=1` |
| 2 | `b` | 括号外 | `len=2` |
| 3 | `_` | 括号外 | 单词 `ab` 结束，`mx=2`，`len=0` |
| 4 | `c` | 括号外 | `len=1` |
| 5 | `d` | 括号外 | `len=2` |
| 6 | `(` | 括号外 -> 括号内 | 先结算 `cd`，再令 `in=true` |
| 7 | `e` | 括号内 | `len=1` |
| 8 | `f` | 括号内 | `len=2` |
| 9 | `_` | 括号内 | 单词 `ef` 结束，`cnt=1` |
| 10 | `g` | 括号内 | `len=1` |
| 11 | `)` | 括号内 -> 括号外 | 先结算 `g`，`cnt=2`，再令 `in=false` |
| 12 | `h` | 括号外 | `len=1` |

扫描结束后，还要结算最后的 `h`。

教师要强调：

遇到 `(` 或 `)` 时，必须先结算当前单词，再改变括号状态。

### 5.4 723B 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

bool isletter(char c){
    // 题目中的单词可能含大小写字母
    return (c >= 'a' && c <= 'z') || (c >= 'A' && c <= 'Z');
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    string s;
    cin >> n >> s;

    bool in = false; // false 表示括号外，true 表示括号内
    int len = 0;     // 当前连续字母段长度
    int mx = 0;      // 括号外最长单词长度
    int cnt = 0;     // 括号内单词个数

    auto finish = [&](){
        if(len == 0) return; // 没有正在统计的单词，不需要结算

        if(in) cnt++;        // 括号内：统计单词个数
        else mx = max(mx, len); // 括号外：更新最长长度

        len = 0;             // 结算后清空当前单词长度
    };

    for(char c: s){
        if(isletter(c)){
            len++;           // 字母属于当前单词
        }else{
            finish();        // 分隔符会结束当前单词

            if(c == '(') in = true;  // 进入括号
            if(c == ')') in = false; // 离开括号
        }
    }

    finish(); // 字符串末尾可能还有一个单词

    cout << mx << ' ' << cnt << '\n';
    return 0;
}
```

### 5.5 讲题时的易错点

- 忘记最后 `finish()`，导致末尾单词漏算。
- 遇到括号后先改变 `in`，导致括号前的单词被算到错误区域。
- 把 `_` 和括号都当作分隔符，但没有区分括号状态变化。
- 字母判断只写小写，漏掉大写。

## 6. 模型二：通配符匹配

### 6.1 一句话核心

只有一个 `*` 的通配符匹配，不需要复杂字符串算法。把模式串按 `*` 拆成前缀和后缀即可。

### 6.2 题目模型：1023A

模式串 `p` 可能含有一个 `*`。

`*` 可以匹配任意长度字符串，包括空串。

例：

```text
p = ab*cd
t = abxyzcd
```

拆分：

```text
pre = ab
suf = cd
```

只要原串开头是 `ab`，结尾是 `cd`，中间 `xyz` 就交给 `*`。

### 6.3 分情况讨论

情况一：没有 `*`

```text
p 必须和 t 完全相同
```

情况二：有 `*`

设：

```text
pre = * 左边的部分
suf = * 右边的部分
```

必须满足：

1. `t.size() >= pre.size() + suf.size()`；
2. `t` 的开头等于 `pre`；
3. `t` 的结尾等于 `suf`。

为什么要检查长度？

如果：

```text
p = abc*def
t = abcde
```

前缀和后缀总长度是 6，但 `t` 长度只有 5，不可能同时放下。

### 6.4 1023A 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    string p, t;
    cin >> n >> m >> p >> t;

    int pos = -1; // 记录 * 的位置；-1 表示没有 *
    for(int i = 0; i < n; i++){
        if(p[i] == '*') pos = i;
    }

    if(pos == -1){
        // 没有通配符时，必须完全相同
        cout << (p == t ? "YES" : "NO") << '\n';
        return 0;
    }

    string pre = p.substr(0, pos);       // * 左边
    string suf = p.substr(pos + 1);      // * 右边

    // * 可以匹配空串，但原串至少要容纳 pre 和 suf
    if((int)t.size() < (int)pre.size() + (int)suf.size()){
        cout << "NO\n";
        return 0;
    }

    bool ok = true;

    // 检查前缀
    for(int i = 0; i < (int)pre.size(); i++){
        if(t[i] != pre[i]) ok = false;
    }

    // 检查后缀。t 的后缀起点是 t.size() - suf.size()
    int start = (int)t.size() - (int)suf.size();
    for(int i = 0; i < (int)suf.size(); i++){
        if(t[start + i] != suf[i]) ok = false;
    }

    cout << (ok ? "YES" : "NO") << '\n';
    return 0;
}
```

复杂度：`O(n+m)`。

### 6.5 教师提问

问：

```text
p = a*b
t = ab
是否匹配？
```

答：匹配，`*` 匹配空串。

问：

```text
p = a*b
t = a
是否匹配？
```

答：不匹配，因为后缀 `b` 放不下。

## 7. 模型三：前缀和处理多次查询

### 7.1 一句话核心

如果很多次询问都在问区间里的某种数量，先把每个位置的贡献算出来，再做前缀和。

### 7.2 题目模型：313B

给字符串 `s`，多次询问 `[l,r]`，问有多少个位置 `i` 满足：

```text
l <= i < r
s[i] == s[i+1]
```

注意：统计对象不是字符本身，而是相邻位置之间的关系。

### 7.3 手动模拟

令：

```text
s = aabbaa
```

1 下标：

| `i` | 1 | 2 | 3 | 4 | 5 |
|---:|---|---|---|---|---|
| 比较 | `a=a` | `a!=b` | `b=b` | `b!=a` | `a=a` |
| 贡献 `b[i]` | 1 | 0 | 1 | 0 | 1 |
| 前缀 `pre[i]` | 1 | 1 | 2 | 2 | 3 |

询问 `[2,6]`：

要统计的是 `i=2,3,4,5`。

答案：

```text
pre[5] - pre[1] = 3 - 1 = 2
```

### 7.4 为什么是 `r-1`

区间 `[l,r]` 中能比较的相邻对是：

```text
(l,l+1), (l+1,l+2), ..., (r-1,r)
```

最后一个左端点是 `r-1`，不是 `r`。

所以公式是：

```cpp
ans = pre[r - 1] - pre[l - 1];
```

### 7.5 313B 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

const int N = 100000 + 10;
int pre[N]; // pre[i] 表示 1..i 中有多少个位置满足 s[j]==s[j+1]

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;

    int n = (int)s.size();
    s = " " + s; // 转成 1 下标，原来的 s[0] 变成 s[1]

    for(int i = 1; i < n; i++){
        int add = (s[i] == s[i + 1]); // 位置 i 和 i+1 是否相等
        pre[i] = pre[i - 1] + add;
    }

    int q;
    cin >> q;
    while(q--){
        int l, r;
        cin >> l >> r;

        // 只统计左端点 i=l..r-1 的相邻对
        cout << pre[r - 1] - pre[l - 1] << '\n';
    }

    return 0;
}
```

复杂度：预处理 `O(n)`，每个询问 `O(1)`，总复杂度 `O(n+q)`。

### 7.6 教师强调

这题最适合训练“统计对象”。

不要只说“前缀和”，要让学生说清楚：

```text
pre[i] 统计的是前 i 个相邻位置左端点的贡献。
```

如果学生说不清楚 `pre[i]` 的含义，后面做 DP 和数据结构会很吃力。

## 8. 模型四：bitmask 表示小集合

### 8.1 一句话核心

如果集合元素很少，可以用二进制位表示选了哪些元素。

对于 `A/B/C` 三种维生素：

| 维生素 | 二进制位 | 十进制值 |
|---|---:|---:|
| `A` | 第 0 位 | 1 |
| `B` | 第 1 位 | 2 |
| `C` | 第 2 位 | 4 |

因此：

| 字符串 | 二进制 | 十进制 mask |
|---|---|---:|
| `A` | `001` | 1 |
| `B` | `010` | 2 |
| `C` | `100` | 4 |
| `AB` | `011` | 3 |
| `BC` | `110` | 6 |
| `ABC` | `111` | 7 |

### 8.2 常用位运算

| 写法 | 含义 |
|---|---|
| `m |= 1` | 加入 `A` |
| `m |= 2` | 加入 `B` |
| `m |= 4` | 加入 `C` |
| `a | b` | 合并两个集合 |
| `m == 7` | 已经包含 `A/B/C` |
| `m & 2` | 判断是否含有 `B` |

教师解释：

`|` 是“有就保留”。只要某个集合里有这一位，合并后就有这一位。

### 8.3 题目模型：1042B

有若干瓶饮料，每瓶有价格和包含的维生素组合。问最小花费，使得买到的维生素包含 `A/B/C`。

朴素想法：枚举买哪些瓶，复杂度 `O(2^n)`，不行。

优化观察：

真正重要的不是“买了哪瓶”，而是“当前集合 mask 是多少”。`A/B/C` 只有 3 种，所以 mask 只有 8 种。

可以先记录每种 mask 的最低价格：

```text
cost[mask] = 买到恰好这个维生素集合的一瓶饮料的最低价格
```

然后枚举最多 3 瓶的组合即可。因为总共只有 3 种维生素，最优方案最多可以用 3 瓶分别补齐。

注意：为了允许“只买 1 瓶”或“只买 2 瓶”的情况，要把 `cost[0]=0`，表示某个枚举位置不买任何饮料。

### 8.4 手动模拟

| 价格 | 维生素 | mask |
|---:|---|---:|
| 5 | `A` | 1 |
| 6 | `BC` | 6 |
| 10 | `AB` | 3 |

枚举组合：

| 选择 | 合并 mask | 费用 |
|---|---:|---:|
| `A + BC` | `1 | 6 = 7` | 11 |
| `BC + AB` | `6 | 3 = 7` | 16 |
| `A + AB` | `1 | 3 = 3` | 不完整 |

答案是 11。

### 8.5 1042B 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

const int INF = 1e9;

int getmask(string s){
    int m = 0;
    for(char c: s){
        if(c == 'A') m |= 1; // 第 0 位表示 A
        if(c == 'B') m |= 2; // 第 1 位表示 B
        if(c == 'C') m |= 4; // 第 2 位表示 C
    }
    return m;
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> cost(8, INF); // 只有 0..7 八种集合状态
    cost[0] = 0;              // 空集合，表示不买，用来处理只买 1 瓶或 2 瓶的情况

    for(int i = 1; i <= n; i++){
        int c;
        string s;
        cin >> c >> s;

        int m = getmask(s);
        cost[m] = min(cost[m], c); // 同一种集合只保留最便宜的一瓶
    }

    int ans = INF;

    // 枚举最多三瓶的集合状态。8^3 很小，可以直接枚举。
    for(int a = 0; a < 8; a++){
        for(int b = 0; b < 8; b++){
            for(int c = 0; c < 8; c++){
                if(cost[a] == INF || cost[b] == INF || cost[c] == INF) continue;

                int all = a | b | c; // 合并三瓶的维生素集合
                if(all == 7){
                    ans = min(ans, cost[a] + cost[b] + cost[c]);
                }
            }
        }
    }

    if(ans == INF) cout << -1 << '\n';
    else cout << ans << '\n';

    return 0;
}
```

复杂度：`O(n + 8^3)`，可以看作 `O(n)`。

### 8.6 可以补充的 DP 写法

如果班里学生基础较好，可以补充一个更通用的状态转移：

```cpp
vector<int> dp(8, INF);
dp[0] = 0;

for(每一瓶饮料){
    int m = getmask(s);
    for(int old = 0; old < 8; old++){
        int nxt = old | m;
        dp[nxt] = min(dp[nxt], dp[old] + c);
    }
}
```

注意：如果要严格避免同一瓶重复使用，应该用临时数组或从旧状态转移。本题用枚举 3 瓶的写法更直观。

## 9. 模型五：二维字符统计

### 9.1 一句话核心

如果每个格子都要判断“它在所在行/列中是否唯一”，不要每次重新扫行列。先统计，再筛选。

### 9.2 题目模型：90B

给 `n*m` 字符矩阵，输出所有满足下面条件的字符：

```text
该字符在它所在行只出现一次；
该字符在它所在列也只出现一次。
```

按原矩阵从上到下、从左到右输出。

### 9.3 设计变量

| 变量 | 含义 |
|---|---|
| `a[i][j]` | 第 `i` 行第 `j` 列字符 |
| `row[i][c]` | 第 `i` 行字符 `c` 出现次数 |
| `col[j][c]` | 第 `j` 列字符 `c` 出现次数 |

### 9.4 为什么要两遍扫描

第一遍：读入并统计。

第二遍：判断每个格子是否保留。

如果对每个格子都重新扫一遍行和列，复杂度大约是：

```text
O(nm(n+m))
```

而两遍统计是：

```text
O(nm)
```

### 9.5 90B 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

const int N = 110;

char a[N][N];
int row[N][26]; // row[i][x]：第 i 行字母 x 出现次数
int col[N][26]; // col[j][x]：第 j 列字母 x 出现次数

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    for(int i = 1; i <= n; i++){
        for(int j = 1; j <= m; j++){
            cin >> a[i][j];

            int x = a[i][j] - 'a';
            row[i][x]++;
            col[j][x]++;
        }
    }

    string ans;

    for(int i = 1; i <= n; i++){
        for(int j = 1; j <= m; j++){
            int x = a[i][j] - 'a';

            // 行里唯一，并且列里唯一，才能保留
            if(row[i][x] == 1 && col[j][x] == 1){
                ans += a[i][j];
            }
        }
    }

    cout << ans << '\n';
    return 0;
}
```

### 9.6 易错点

- 输出顺序是原矩阵顺序，不是排序后输出。
- 行统计和列统计要分开。
- 字符映射 `a[i][j]-'a'` 默认题目字符是小写字母。

## 10. 模型六：多关键字排序与并列统计

### 10.1 一句话核心

排序题先把“谁排前面”写成比较函数，再在排序后的数组里定位目标。

### 10.2 题目模型：166A

给 `n` 个队伍，每个队伍有：

- 解题数 `p`；
- 罚时 `t`。

排名规则：

1. `p` 越大排名越靠前；
2. `p` 相同，`t` 越小排名越靠前。

问：和第 `k` 名成绩完全相同的队伍有多少个。

### 10.3 排序后统计

步骤：

1. 按规则排序。
2. 取出排序后第 `k` 个队伍的 `(p,t)`。
3. 扫一遍数组，统计有多少队伍 `(p,t)` 完全相同。

注意：

这里不是问“第 k 名所在排名段有几个”，但效果等价于统计同样 `(p,t)` 的队伍数量。

### 10.4 166A 参考代码

```cpp
#include<bits/stdc++.h>
using namespace std;

struct Node{
    int p, t; // p: solved problems, t: penalty time
};

bool cmp(Node a, Node b){
    if(a.p != b.p) return a.p > b.p; // 解题数多的排前面
    return a.t < b.t;                // 解题数相同，罚时少的排前面
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    cin >> n >> k;

    vector<Node> a(n + 1); // 使用 1 下标，方便直接访问 a[k]
    for(int i = 1; i <= n; i++){
        cin >> a[i].p >> a[i].t;
    }

    sort(a.begin() + 1, a.end(), cmp);

    Node key = a[k]; // 第 k 名的成绩
    int ans = 0;

    for(int i = 1; i <= n; i++){
        if(a[i].p == key.p && a[i].t == key.t){
            ans++;
        }
    }

    cout << ans << '\n';
    return 0;
}
```

复杂度：排序 `O(n log n)`，统计 `O(n)`。

### 10.5 教师强调比较函数

错误写法示例：

```cpp
return a.p >= b.p;
```

这个写法不合法，因为当 `a.p == b.p` 时，`cmp(a,b)` 和 `cmp(b,a)` 都可能为真。排序比较函数要表达严格的“排在前面”，不能用 `>=` 或 `<=`。

### 12.1 判断复杂度

```cpp
for(int i = 1; i <= n; i++){
```

### 12.2 判断前缀和公式

给：

```text
s = aabbaa
查询 [2,6]
```

原因：最后比较的是 `s[5]` 和 `s[6]`。

### 12.3 判断 bitmask

```text

BC = 2 | 4 = 6
3 | 6 = 7
```

说明已经凑齐 `ABC`。

## 14. 本课总易错点清单

### 14.1 复杂度

- 只看循环层数，不看变量变化。
- 忘记多组数据的总规模。
- 把 `sort` 当成 `O(n)`。

### 14.2 字符串

- `string` 是 0 下标，前缀和经常转 1 下标。
- `substr(pos, len)` 第二个参数是长度。
- 扫描结束后忘记结算最后一段。
- `*` 可以匹配空串。

### 14.3 前缀和

- 没说清楚 `pre[i]` 统计什么。
- 相邻关系题统计到 `r-1`，不是 `r`。
- `pre[0]` 要有意义。

### 14.4 bitmask

- 把 `|` 和 `+` 混用。
- 忘记检查不可达状态 `INF`。
- 不知道 `7` 代表二进制 `111`。

### 14.5 排序

- 比较函数写 `>=` 或 `<=`。
- 多关键字顺序写反。
- 排序后忘记第 `k` 名是 `a[k]` 还是 `a[k-1]`。

## 17. 本节总结

- 数据范围先决定复杂度方向。
- 字符串题优先想“扫描 + 状态 + 结算”。
- 多次区间查询优先想“贡献数组 + 前缀和”。
- 小集合问题优先想 bitmask。
- 排序题先写清楚比较规则，再处理排名或统计。
